# Quantum Language Interpreter

## Overview

The Quantum Language Interpreter is the core execution engine responsible for evaluating the Abstract Syntax Tree (AST) generated by the Quantum Language compiler. It handles the execution of quantum scripts by interpreting each node in the AST according to its type and content. The interpreter supports various operations including variable declarations, function calls, conditional statements, loops, and more.

## Architecture

The interpreter is designed as a tree-walk interpreter, meaning it processes the AST from top to bottom, visiting each node and executing the corresponding operation. The architecture includes several key components:

1. **Statement Execution**: Handles the execution of different types of statements such as `if`, `while`, `for`, and `return`.
2. **Expression Evaluation**: Evaluates expressions to compute their values. This includes handling identifiers, unary operators, assignment expressions, function calls, and more.
3. **Built-in Function Implementations**: Provides implementations for built-in functions used in quantum scripts.
4. **Memory Management**: Manages memory allocation and deallocation for variables and objects within the quantum script environment.

### Key Files and Functions

- **Interpreter.cpp**: Contains the main interpreter logic, including the `execute` function which drives the interpretation process.
- **Visitor.h/Visitor.cpp**: Defines the visitor pattern used to traverse the AST. Each node type has a corresponding visit method.
  - `visit`: A generic visit method that dispatches to specific visit methods based on the node type.
  - Specific visit methods like `execBlock`, `execVarDecl`, etc., handle the execution of different node types.
- **Evaluator.h/Evaluator.cpp**: Implements the expression evaluation logic.
  - `evaluate`: The main entry point for expression evaluation.
  - Specific evaluation methods like `evalIdentifier`, `evalUnary`, `evalAssign`, etc., handle the evaluation of different expression types.
- **FunctionCallHandler.h/FunctionCallHandler.cpp**: Manages the calling of both native and user-defined functions.
  - `callFunction`: Handles the general function call mechanism.
  - `callNative`: Calls native C++ functions.
  - `callInstanceMethod`: Calls instance methods of objects.
- **MemoryManager.h/MemoryManager.cpp**: Manages memory allocation and deallocation.
  - `allocMemory`: Allocates memory for variables and objects.
  - `freeMemory`: Frees allocated memory when it's no longer needed.

## Flow

1. **Compilation**: The Quantum Language compiler generates an AST from the source code.
2. **Interpretation**:
   - The `execute` function starts the interpretation process by calling the appropriate visit method for the root node of the AST.
   - Each visit method then recursively visits child nodes and executes the corresponding operations.
   - Expressions are evaluated using the `evaluate` function, which uses specific evaluation methods depending on the expression type.
   - Function calls are handled by the `FunctionCallHandler`, which determines whether the function is native or user-defined and invokes the appropriate handler.
3. **Execution**: Statements are executed directly, while expressions return their computed values.
4. **Memory Management**: Variables and objects are allocated and freed as they go out of scope, ensuring efficient use of resources.

## Usage

To use the Quantum Language Interpreter, you need to have an AST generated by the Quantum Language compiler. Once you have the AST, you can pass it to the `execute` function to start the interpretation process.

```cpp
#include "Interpreter.h"

int main() {
    // Assume 'ast' is a valid Abstract Syntax Tree generated by the compiler
    ASTNode* ast = ...;

    Interpreter interpreter;
    interpreter.execute(ast);

    return 0;
}
```
