# Quantum Language Compiler - src/Compiler.cpp

## Overview

The `src/Compiler.cpp` file is a crucial component of the Quantum Language compiler, responsible for converting Abstract Syntax Trees (ASTs) into executable bytecode. This process involves several stages, including parsing, semantic analysis, optimization, and code generation. The `Compiler` class encapsulates the logic needed to traverse the AST and generate corresponding bytecode instructions.

## Role in Compiler Pipeline

- **Parsing**: Converts source code into an AST.
- **Semantic Analysis**: Validates the AST and resolves symbols.
- **Code Generation**: Translates the validated AST into bytecode that can be executed by the virtual machine.

## Key Design Decisions and Why

### Global Chunk Registry

- **Design Decision**: Maintains a global map (`g_chunkRegistry`) to store chunks of bytecode associated with native quantum functions.
- **Why**: This allows efficient reuse of compiled code for frequently called quantum operations, reducing memory usage and compilation time.

### CompilerState Class

- **Overview**: Represents the state of the compiler at any given point during the compilation process.
- **Key Components**:
  - `scopeDepth`: Tracks the depth of nested scopes.
  - `locals`: Stores information about local variables within the current scope.
  - `upvalues`: Keeps track of upvalues used in closures.
  - `chunk`: Holds the generated bytecode for the current function or script.

### Scope Management

- **BeginScope**: Increments the scope depth.
- **EndScope**: Decrements the scope depth and pops locals until the current scope depth is reached. If a local is captured by an enclosing scope, it emits a `CLOSE_UPVALUE` instruction; otherwise, it emits a `POP` instruction.

### Symbol Resolution

- **ResolveLocal**: Searches for a variable in the current scope.
- **AddUpvalue**: Adds an upvalue to the current scope's upvalue list.
- **ResolveUpvalue**: Searches for a variable in an enclosing scope and adds it as an upvalue if found.

### Local Variable Declaration

- **DeclareLocal**: Adds a new local variable to the current scope's local variable list.
- **Why**: This ensures that all local variables are properly tracked and managed throughout the compilation process.

### Bytecode Emission

- **EmitLoad**: Emits a load instruction for a local or global variable.
- **EmitStore**: Emits a store instruction for a local or global variable.

## Major Classes/Functions Overview

### Compiler Class

- **Constructor**: Initializes the compiler state with a default script name.
- **Compile**: Main entry point for compiling an AST node. It initializes the compiler state, compiles the root node, and appends a `RETURN_NIL` instruction before returning the compiled chunk.

### CompilerState Struct

- **Scope Depth**: Manages the nesting level of scopes.
- **Locals**: Tracks local variables within each scope.
- **Upvalues**: Handles upvalues in closures.
- **Chunk**: Accumulates the generated bytecode.

### Scope Management Functions

- **BeginScope**: Increases the scope depth.
- **EndScope**: Decreases the scope depth and manages local variable lifetimes.

### Symbol Resolution Functions

- **ResolveLocal**: Locates a local variable within the current scope.
- **AddUpvalue**: Registers an upvalue for use in closures.
- **ResolveUpvalue**: Finds a variable in an enclosing scope and registers it as an upvalue.

### Local Variable Declaration Function

- **DeclareLocal**: Adds a new local variable to the current scope.

### Bytecode Emission Functions

- **EmitLoad**: Generates a load instruction for a variable.
- **EmitStore**: Generates a store instruction for a variable.

## Tradeoffs

- **Memory Usage**: Reusing compiled chunks through the global registry reduces memory consumption but may increase complexity in managing dependencies between chunks.
- **Compilation Time**: Efficiently reusing compiled chunks can significantly reduce compilation time for scripts that call the same quantum operations multiple times.
- **Flexibility**: While reusing compiled chunks improves performance, it limits the flexibility of modifying or replacing individual quantum operations without affecting other parts of the program.
