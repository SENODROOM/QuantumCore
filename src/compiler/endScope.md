# `endScope` Function

## Purpose
The `endScope` function is responsible for managing the scope depth and cleaning up local variables when exiting a scope in the Quantum Language compiler. This ensures that resources associated with local variables are properly released to prevent memory leaks or dangling references.

## Parameters
- None explicitly declared in the provided code snippet.

## Return Value
- The function does not return any value (`void`).

## How It Works
1. **Decrement Scope Depth**: 
   ```cpp
   current_->scopeDepth--;
   ```
   - This line decreases the `scopeDepth` of the current chunk. The `scopeDepth` tracks how many nested scopes are currently active. Decrementing it indicates that one less scope is being exited.

2. **Cleanup Local Variables**:
   ```cpp
   while (!current_->locals.empty() && 
          current_->locals.back().depth > current_->scopeDepth)
   {
       // Cleanup logic here
   }
   ```
   - The function then enters a loop that continues as long as there are local variables (`!current_->locals.empty()`) and the depth of the last local variable (`current_->locals.back().depth`) is greater than the updated `scopeDepth`.
   
3. **Emitting Operations**:
   - Inside the loop, the function checks whether the last local variable was captured using an upvalue (`current_->locals.back().isCaptured`). If it was captured, it emits an `Op::CLOSE_UPVALUE` operation to close the upvalue. This operation ensures that the upvalue is properly closed and cleaned up.
   - If the last local variable was not captured, it emits an `Op::POP` operation to pop the variable off the stack. This operation removes the variable from the stack, effectively freeing its resources.
   
4. **Removing Local Variables**:
   ```cpp
   current_->locals.pop_back();
   ```
   - After emitting the appropriate operation, the function removes the last local variable from the `locals` vector. This step ensures that the list of local variables only contains those that are still within the current scope.

## Edge Cases
- **Empty Locals Vector**: If the `locals` vector is empty at the time `endScope` is called, the function will simply exit without performing any cleanup operations.
- **Local Variable Depth Not Matching**: If the depth of the last local variable does not match the expected `scopeDepth`, the function may behave unpredictably. However, based on the loop condition, this should not occur under normal circumstances.

## Interactions with Other Components
- **Chunk Management**: The `endScope` function interacts with the `Chunk` class through the `current_` pointer. The `Chunk` class likely manages the bytecode instructions and local variables for each scope.
- **Bytecode Emission**: The function uses the `emit` method to add operations to the current chunk's bytecode. This method is crucial for generating the executable bytecode that will be run by the Quantum Virtual Machine.
- **Upvalues**: The `emit` method is also used to handle upvalues. Upvalues are used to capture local variables from outer scopes, and closing them is essential to avoid memory leaks.

Overall, the `endScope` function plays a critical role in ensuring proper resource management and maintaining the correctness of the generated bytecode during compilation.
