# skipComment() Function - Single-Line Comment Skipping

## Overview
The `skipComment()` function is designed to consume and discard single-line comments in the source code that start with `//`. This function ensures that the lexer skips over these comments without including them in the token stream, thereby preserving the integrity of the parsing process.

### Parameters
- None

### Return Value
- None

### Edge Cases
1. **Empty Line**: If the line immediately following `//` is empty, the function will correctly identify it as a comment and skip to the next line.
2. **End-of-File**: If the file ends before encountering a newline character after `//`, the function will stop at the end of the file.
3. **Multi-Line Comments**: The function only handles single-line comments. Multi-line comments starting with `/* */` are not processed by this method.

### Interactions with Other Components
- **Lexer Class**: The `skipComment()` function is part of the Lexer class, which is responsible for breaking down the source code into tokens.
- **Tokenizer**: After calling `skipComment()`, the tokenizer can safely proceed to the next token without being affected by the comment content.
- **Error Handling**: Although not explicitly shown in the provided code snippet, any issues encountered during the skipping process (e.g., unexpected characters) should be handled appropriately within the Lexer or Tokenizer class.

### Implementation Details
The implementation of `skipComment()` uses a simple loop to iterate through the source code until it encounters a newline character (`'\n'`). Here’s how it works:

```cpp
void Lexer::skipComment() {
    // Loop until the end of the file or a newline character is found
    while (pos < src.size() && current() != '\n') {
        // Advance the lexer position to the next character
        advance();
    }
}
```

#### Why It Works This Way
- **Character Iteration**: By iterating through each character using `advance()`, the function ensures that it processes every character in the comment line.
- **Newline Detection**: The loop continues until a newline character is encountered, indicating the end of the comment line.
- **Efficiency**: This approach is efficient because it stops processing as soon as the end of the comment line is reached, without unnecessary checks or operations.

This function is crucial for maintaining the cleanliness and accuracy of the token stream generated by the lexer, ensuring that comments do not interfere with the parsing logic of the quantum language compiler.
