# QuantumLanguage Compiler - Vm.h

## Overview

The `include/Vm.h` header file is an essential part of the QuantumLanguage compiler, focusing on the virtual machine (VM) that executes compiled code. It defines structures and classes crucial for managing execution contexts, handling exceptions, and maintaining the runtime environment. The VM plays a pivotal role in interpreting and executing the bytecode generated by the compiler.

## Key Design Decisions and Why

### Memory Management with Smart Pointers

To ensure efficient memory management and prevent memory leaks, the VM uses smart pointers (`std::shared_ptr`) extensively. This approach simplifies resource cleanup and reduces the risk of dangling references.

### Efficient Data Structures for Runtime Environment

- **Stack**: Implemented as a `std::vector`, providing fast access and modification operations.
- **Frames**: Also managed using a `std::vector`, each representing a function call's context.
- **Handlers**: An `std::vector` of `ExceptionHandler` structs allows for quick lookup and unwinding during exception handling.

### Trade-offs Considered

- **Memory Overhead**: Using smart pointers introduces some overhead due to reference counting. However, it significantly improves safety and reduces manual memory management errors.
- **Performance**: While smart pointers add overhead, the performance impact is generally negligible compared to the benefits they provide in terms of code maintainability and reliability.

## Major Classes/Functions Overview

### Upvalue

**Role**: Represents a heap cell capturing a local variable from an enclosing function scope.

**Key Functions**:
- `get()`: Retrieves the current value of the captured variable.
- `set(QuantumValue v)`: Updates the captured variable with a new value.

### Closure

**Role**: Encapsulates a function along with its upvalues, forming a complete executable unit.

**Key Functions**:
- Constructor: Initializes the closure with a chunk and sets the function name.

### CallFrame

**Role**: Holds information about the current function call, including the closure, instruction pointer, and stack base.

**Key Functions**:
- Constructor: Sets up the initial call frame with a closure and starting positions.

### ExceptionHandler

**Role**: Defines how the VM should handle exceptions, including the target IP and the depths to unwind and restore.

**Key Functions**:
- Constructor: Initializes the handler with specific parameters for exception handling.

### VM Class

**Overview**: Manages the overall execution of the program, including the main loop, stack operations, and function calls.

**Key Functions**:
- `run(std::shared_ptr<Chunk> chunk)`: Executes a top-level script represented by a chunk of bytecode.
- `push(QuantumValue v)`: Adds a value to the stack.
- `pop()`: Removes the top value from the stack.
- `callValue(QuantumValue callee, int argCount, int line)`: Handles calling a generic value (function or method).
- `registerNatives()`: Registers native functions into the VM's global environment.

### Trade-offs in VM Implementation

- **Flexibility vs. Performance**: The use of smart pointers offers flexibility but may introduce slight performance overhead. Balancing these factors ensures that the VM remains both safe and efficient.
- **Complexity**: Managing multiple data structures such as stacks, frames, and handlers adds complexity to the implementation. However, this complexity is necessary for handling complex scenarios like nested function calls and exception propagation.

This file forms the backbone of the QuantumLanguage compiler's runtime system, ensuring that the interpreter can execute code correctly while managing resources efficiently.
