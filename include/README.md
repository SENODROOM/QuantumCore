# QuantumLanguage Compiler - Parser.h

## Overview

The `include/Parser.h` header file is a critical component of the QuantumLanguage compiler, responsible for parsing the source code into an Abstract Syntax Tree (AST). This file encapsulates the logic necessary to convert a sequence of tokens generated by the lexer into a structured representation of the program's syntax. The parser ensures that the code adheres to the language's grammar rules and constructs the appropriate AST nodes based on the parsed input.

### Role in Compiler Pipeline

The parser operates as a central step in the compiler pipeline following the lexical analysis phase. After tokens have been identified and categorized, the parser takes these tokens and organizes them into a hierarchical structure known as an AST. This AST serves as the foundation for subsequent phases such as semantic analysis, optimization, and code generation.

### Key Design Decisions and Why

#### Pratt Parsing Algorithm

The parser uses the Pratt parsing algorithm to handle operator precedence and associativity. This algorithm allows for flexible parsing without requiring lookahead beyond one token, making it efficient and easy to implement. By using Pratt parsing, the compiler can correctly interpret expressions like `a + b * c`, which would otherwise be ambiguous without proper precedence rules.

#### Error Handling

A custom `ParseError` class is defined to handle errors during parsing. This class inherits from `std::runtime_error` but adds additional information about the error location (`line` and `col`). This makes it easier to diagnose issues directly within the source code.

#### Recursive Descent Parsing

The parser employs a recursive descent approach, where each syntactic construct corresponds to a function. This method simplifies the implementation of complex grammars and provides clear, readable code. Each function handles parsing a specific type of construct, such as variable declarations or control flow statements.

### Major Classes/Functions Overview

#### Parser Class

The `Parser` class is the primary entry point for parsing. It initializes with a vector of tokens and provides a `parse()` method to generate the AST. The class contains several private member functions that perform the actual parsing:

- **parseStatement()**: Parses a single statement.
- **parseBlock()**: Parses a block of statements enclosed in curly braces.
- **parseVarDecl()**: Parses variable declarations.
- **parseFunctionDecl()**: Parses function declarations.
- **parseClassDecl()**: Parses class declarations.
- **parseIfStmt()**, **parseWhileStmt()**, **parseForStmt()**: Parses conditional and loop statements.
- **parseReturnStmt()**: Parses return statements.
- **parseExprStmt()**: Parses expression statements.
- **parseExpr()**: Parses expressions using Pratt parsing.
- **parseAssignment()**: Parses assignment expressions.
- **parsePrimary()**: Parses primary expressions (e.g., literals, identifiers).

#### Token Helpers

Several helper functions manage the current position in the token stream:

- **current()**: Returns the current token.
- **peek(int offset)**: Returns the token at a specified offset ahead in the stream.
- **consume()**: Consumes the current token and advances the position.
- **expect(TokenType t, const std::string &msg)**: Ensures the next token matches the expected type, throwing an error if not.
- **check(TokenType t)**: Checks if the next token matches the specified type.
- **match(TokenType t)**: Matches the next token if it matches the specified type, advancing the position if successful.
- **atEnd()**: Determines if the end of the token stream has been reached.
- **skipNewlines()**: Skips any newline tokens.

#### Additional Features

- **parseArrayLiteral()** and **parseDictLiteral()**: Handle array and dictionary literal constructions.
- **parseLambda()** and **parseArrowFunction()**: Support lambda and arrow function syntax.
- **parseArgList()** and **parseParamList()**: Manage argument and parameter lists for functions.

### Tradeoffs

#### Complexity vs. Simplicity

Using a recursive descent parser with Pratt parsing for operator precedence adds complexity to the implementation but results in more readable and maintainable code compared to other approaches like LL(k) parsers.

#### Error Reporting

The custom `ParseError` class provides detailed error reporting, which enhances debugging but may increase compile time slightly due to the overhead of tracking error locations.

#### Flexibility vs. Performance

The Pratt parsing algorithm offers flexibility in handling operator precedence but may introduce slight performance overhead compared to simpler parsing techniques.

Overall, the `Parser.h` file is well-designed to balance readability, flexibility, and robust error handling while efficiently converting source code into an AST.
