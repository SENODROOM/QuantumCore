# TypeChecker.h

This header file is part of the Quantum Language compiler's core functionality, specifically designed to handle static type checking during the compilation process. The `TypeChecker` class plays a crucial role in ensuring that the code adheres to the language's typing rules before any further processing occurs.

## Key Design Decisions

### Use of Exceptions for Error Handling
- **Why**: Exceptions provide a clear and structured way to handle errors, making the code more readable and maintainable. By throwing `StaticTypeError`, we can easily propagate error information up the call stack without cluttering the normal control flow.
  
### Shared Pointer for Type Environment
- **Why**: Using `std::shared_ptr` for managing the type environment allows for efficient memory management and easy sharing between different parts of the compiler. This approach avoids manual memory handling and reduces the risk of memory leaks.

### Hierarchical Type Environment
- **Why**: A hierarchical type environment (`TypeEnv`) enables the compiler to manage variable scopes effectively. Each scope has its own environment, which inherits from its parent, allowing variables defined in outer scopes to be accessible within inner scopes.

## Classes and Functions Documentation

### Class: TypeChecker

**Purpose**: The primary class responsible for performing static type checking on the Abstract Syntax Tree (AST) generated by the parser.

**Behavior**:
- Initializes a new instance of the `TypeChecker` class with an empty global type environment.
- Provides methods to check entire ASTs and individual nodes for type correctness.

### Function: TypeChecker::check(const std::vector<ASTNodePtr>& nodes)

**Purpose**: Checks a vector of AST nodes for type correctness.

**Parameters**:
- `nodes`: A vector of shared pointers to AST nodes representing the program to be checked.

**Returns**: None. Throws exceptions if type errors are found.

### Function: TypeChecker::check(const ASTNodePtr& node)

**Purpose**: Checks a single AST node for type correctness.

**Parameters**:
- `node`: A shared pointer to an AST node representing the code segment to be checked.

**Returns**: None. Throws exceptions if type errors are found.

### Function: TypeChecker::checkNode(const ASTNodePtr& node, std::shared_ptr<TypeEnv> env)

**Purpose**: Recursively checks an AST node for type correctness within a given type environment.

**Parameters**:
- `node`: A shared pointer to an AST node representing the code segment to be checked.
- `env`: A shared pointer to the current type environment.

**Returns**:
- A string representing the type of the node if it is correctly typed.
- Throws `StaticTypeError` if a type error is detected.

### Struct: TypeEnv

**Purpose**: Represents a type environment, which maps variable names to their types and supports hierarchical scoping.

**Fields**:
- `vars`: A map storing variable names as keys and their corresponding types as values.
- `parent`: A shared pointer to the parent type environment, forming a hierarchy.

**Methods**:
- `define(const std::string& name, const std::string& type)`: Defines a new variable in the current environment.
- `resolve(const std::string& name)`: Resolves the type of a variable by searching through the current and parent environments. Returns `"any"` if the variable is not found.

## Tradeoffs and Limitations

- **Performance**: While exceptions are generally slower than returning error codes, they offer significant advantages in terms of clarity and ease of use. For performance-critical applications, alternative approaches such as using error codes might be considered.
  
- **Memory Management**: Using `std::shared_ptr` simplifies memory management but may introduce overhead due to reference counting. In scenarios where memory efficiency is critical, manual memory management techniques could be used instead.

- **Scalability**: The hierarchical type environment model works well for small to medium-sized programs but may become less efficient for very large programs due to increased complexity in managing references.

## Usage Example

```cpp
#include "TypeChecker.h"

int main() {
    // Create an instance of the TypeChecker
    TypeChecker checker;

    // Parse your AST here
    std::vector<ASTNodePtr> astNodes;

    try {
        // Check the entire AST
        checker.check(astNodes);
        std::cout << "All types are correct!" << std::endl;
    } catch (const StaticTypeError& e) {
        std::cerr << "Type error at line " << e.line << ": " << e.what() << std::endl;
    }

    return 0;
}
```

This example demonstrates how to create an instance of `TypeChecker`, parse an AST, and perform type checking. If any type errors are encountered, they will be caught and reported.
