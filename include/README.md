# QuantumLanguage Compiler - Parser.h

## Overview

The `include/Parser.h` header file is an essential part of the QuantumLanguage compiler, focusing on the parsing phase of the compilation process. This file defines the `Parser` class, which takes a vector of `Token`s generated by the lexer and constructs an Abstract Syntax Tree (AST) representing the syntactic structure of the program. The parser handles the conversion of the token stream into a structured tree, enabling subsequent phases of the compiler to perform semantic analysis, optimization, and code generation.

## Key Design Decisions

### Error Handling

**Why:** To provide precise error messages indicating the location and nature of syntax errors, facilitating easier debugging and maintenance.

**Implementation:** A custom `ParseError` class derived from `std::runtime_error` includes line and column numbers, enhancing error reporting.

### Use of `std::variant`

**Why:** To support multiple data types within expressions and variables, ensuring flexibility and robustness in handling different kinds of values.

**Implementation:** Utilizing `std::variant` allows the parser to encapsulate various value types within a single AST node, simplifying the representation and manipulation of these types.

### Pratt Parsing Algorithm

**Why:** To enable flexible and extensible expression parsing, supporting both infix and prefix operators without requiring significant changes to the grammar.

**Implementation:** The Pratt parsing algorithm is employed to handle operator precedence and associativity, allowing the parser to correctly interpret complex expressions.

## Classes and Functions Documentation

### ParseError Class

**Purpose:** Represents a parsing error, providing detailed information about the error message and its location in the input source.

**Behavior:** Inherits from `std::runtime_error` and adds line and column attributes to store error positions.

### Parser Class

**Purpose:** Converts a sequence of tokens into an AST, representing the syntactic structure of the program.

**Behavior:**
- **Constructor:** Initializes the parser with a vector of tokens.
- **parse():** Main entry point for parsing, constructs the AST by calling other parsing methods.
- **Token Helpers:**
  - `current()`: Retrieves the current token being processed.
  - `peek(int offset = 1)`: Looks ahead at a specified number of tokens without advancing the position.
  - `consume()`: Advances the position and returns the current token.
  - `expect(TokenType t, const std::string &msg)`: Consumes the next token if it matches the expected type, otherwise throws a `ParseError`.
  - `check(TokenType t) const`: Checks if the next token matches the specified type.
  - `match(TokenType t)`: Consumes the next token if it matches the specified type.
  - `atEnd() const`: Determines if the end of the token stream has been reached.
  - `skipNewlines()`: Skips over any newline characters during parsing.
- **Parsing Methods:**
  - `parseStatement()`, `parseBlock()`, `parseBodyOrStatement()`: Handle different types of statements and blocks.
  - `parseVarDecl(bool isConst)`, `parseFunctionDecl()`, `parseClassDecl()`: Construct declarations for variables, functions, and classes.
  - `parseIfStmt()`, `parseWhileStmt()`, `parseForStmt()`: Parse control flow structures like if, while, and for loops.
  - `parseReturnStmt()`, `parsePrintStmt()`, `parseInputStmt()`: Handle return, print, and input statements.
  - `parseCoutStmt()`, `parseCinStmt()`: Manage output and input operations using streams.
  - `parseImportStmt(bool isFrom = false)`: Parses import statements, optionally specifying a module name.
  - `parseExprStmt()`: Handles expression statements.
  - `parseCTypeVarDecl(const std::string &typeHint)`: Parses variable declarations with optional type hints.
  - `isCTypeKeyword(TokenType t) const`: Checks if a given token is a keyword related to C-type declarations.
- **Expression Parsing:**
  - `parseExpr()`, `parseAssignment()`, `parseOr()`, `parseAnd()`, etc.: Implement Pratt-style parsing for different levels of operator precedence and associativity.
  - `parseArrayLiteral()`, `parseDictLiteral()`: Construct AST nodes for array and dictionary literals.
  - `parseLambda()`, `parseArrowFunction(std::vector<std::string> params, int ln)`: Handle lambda functions and arrow functions.
  - `parseArgList()`, `parseParamList(std::vector<bool> *outIsRef = nullptr, std::vector<ASTNodePtr> *outDefaultArgs = nullptr, std::vector<std::string> *outParamTypes = nullptr)`: Parse argument lists and parameter lists, including reference parameters and default arguments.

## Tradeoffs and Limitations

- **Flexibility vs. Complexity:** While the use of `std::variant` enhances flexibility, it also increases the complexity of the AST and the associated code.
- **Performance:** Pratt parsing is efficient but may not scale well with extremely large programs due to its recursive nature.
- **Error Reporting:** Precise error messages require careful tracking of token positions, which can add overhead to the parsing process.
- **Extensibility:** Adding new language features requires updates to the parser and potentially the entire AST system.

This file serves as the foundation for the QuantumLanguage compiler's parsing capabilities, ensuring that the syntactic structure of the program is accurately represented and ready for further processing.
