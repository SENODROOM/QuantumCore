# QuantumLanguage Compiler - Compiler.h

## Overview

The `include/Compiler.h` header file is a critical component of the QuantumLanguage compiler, specifically dedicated to the compilation process. This file contains essential structures and classes necessary for parsing abstract syntax trees (ASTs), generating bytecode, and managing the compilation context. The primary focus is on the `Compiler` class, which orchestrates the compilation of quantum programs into executable chunks.

## Role in Compiler Pipeline

In the QuantumLanguage compiler pipeline, the `Compiler.h` file serves as the heart of the compilation phase. After the AST is generated by the parser, the `Compiler` class takes over to translate the AST into bytecode that can be executed by the virtual machine (VM). This includes handling variable declarations, function definitions, control flow statements, and other constructs specific to the QuantumLanguage.

## Key Design Decisions and Why

### Virtual Machine Integration

The decision to integrate a virtual machine within the compiler was made to ensure efficient execution of the compiled code. By compiling into bytecode and running it through a VM, the compiler can leverage optimizations available at the VM level, such as just-in-time (JIT) compilation and garbage collection.

### Scoped Variable Management

To manage variables effectively, especially considering nested scopes and closures, the compiler uses a scoped variable system. Each scope maintains a list of local variables and their indices. This allows for quick resolution of variable references and promotes better memory usage by minimizing stack space allocation.

### Dynamic Memory Allocation

Using smart pointers (`std::shared_ptr`) for managing memory ensures automatic deallocation when objects are no longer needed. This reduces the risk of memory leaks and simplifies resource management without manual intervention.

## Major Classes/Functions Overview

### Compiler Class

- **Overview**: The main class responsible for driving the compilation process. It manages the current state of the compilation, including the chunk of bytecode being generated and the scope of variables.
- **Key Functions**:
  - `compile(ASTNode &root)`: Compiles an entire program starting from the root AST node.
  - `beginScope()` and `endScope(int line = 0)`: Manage the beginning and end of variable scopes, respectively.
  - `resolveLocal(CompilerState *state, const std::string &name)` and `resolveUpvalue(CompilerState *state, const std::string &name)`: Resolve variable references based on their scope and whether they are captured by closures.
  - `declareLocal(const std::string &name, int line = 0)`: Declare a new local variable and assign it a unique index.

### Helper Functions

- **Overview**: These functions provide utility for emitting bytecode instructions, adding constants and strings to the chunk, and managing jumps and labels.
- **Key Functions**:
  - `emit(Op op, int32_t operand = 0, int line = 0)`: Emits a single bytecode instruction with optional operand and line number.
  - `addConst(QuantumValue v)` and `addStr(const std::string &s)`: Add constants and strings to the chunk and return their respective indices.
  - `emitJump(Op op, int line = 0)`: Emits a jump instruction with a placeholder operand.
  - `patchJump(size_t idx)`: Patches the operand of a jump instruction to reflect the actual offset in the bytecode.

### Statement Compilation Functions

- **Overview**: These functions handle the compilation of different types of statements found in the AST.
- **Key Functions**:
  - `compileVarDecl(VarDecl &s, int line)`: Compiles a variable declaration statement.
  - `compileFunctionDecl(FunctionDecl &s, int line)`: Compiles a function declaration statement.
  - `compileClassDecl(ClassDecl &s, int line)`: Compiles a class declaration statement.
  - `compileIf(IfStmt &s, int line)`: Compiles an if-statement, including conditional branching and jump patching.

## Tradeoffs

### Performance vs. Simplicity

Integrating a VM adds complexity but also offers significant performance benefits. While the initial implementation might be more intricate, the long-term efficiency gains justify the added overhead.

### Memory Usage

Scoped variable management helps reduce memory usage by reusing slots where possible. However, this requires careful tracking of variable lifetimes and scopes, which can introduce additional complexity.

### Flexibility vs. Consistency

Allowing dynamic memory allocation via smart pointers provides flexibility in managing resources. On the other hand, this can lead to potential issues if not handled carefully, such as premature deallocation or memory fragmentation.

Overall, the `include/Compiler.h` header file plays a vital role in the QuantumLanguage compiler by providing the necessary tools and mechanisms for translating high-level quantum programs into low-level bytecode that can be efficiently executed by a VM. Its design decisions balance performance, simplicity, memory usage, and flexibility, making it a robust foundation for the compiler's functionality.
